# 代理式流量 {#agentic-traffic}

代理式流量儀表板會顯示 AI 代理 (爬蟲和聊天機器人) 如何與您的網站互動。 您可以利用這個視圖來追蹤要求的總數以及一般效能相關量度。 您也可以檢視橫跨多個市場、類別、頁面和代理的流量分佈情形。 此儀表板使用的資料取自內容傳遞網路記錄，因此您必須設定&#x200B;**內容傳遞網路記錄轉送**&#x200B;才能顯示量度。 您也可以利用自訂篩選器來精確調整所顯示的資料。 導覽至&#x200B;**代理流量**，並選取您要檢視代理流量深入分析的網站。

## 內容傳遞網路記錄轉送 {#cdn-setup}

如果沒有&#x200B;**內容傳遞網路記錄轉送**，則代理式流量儀表板是空白的。 若要檢視代理式互動，您必須設定&#x200B;**內容傳遞網路記錄檔轉送**。

您可以瀏覽至&#x200B;**品牌管理**&#x200B;並按一下&#x200B;**CDN**&#x200B;標籤，新增CDN記錄轉送資訊。

![品牌管理 — 內容傳遞網路記錄轉送](/help/assets/brand-centric-experience/brands-management-cdn.png)

關於使用客戶管理的內容傳遞網路 (BYOCDN) 時進行記錄轉送的詳細資訊，請參閱 [BYOCDN 記錄轉送概觀](/help/overview/log-forwarding/log-forwarding-overview.md)

LLM Optimizer 處理內容傳遞網路記錄中的一部分欄位。 雖然原始記錄欄位名稱會因內容傳遞網路提供者而異，這些名稱經過標準化後，呈現如下：

* URL (路徑和查詢參數)
* 使用者代理
* 狀態代碼
* 反向連結標頭
* 主機標頭
* 第一個位元組時間 (TTFB)
* 要求方法
* 時間戳記
* 內容類型

這些標準化的欄位會透過代理式視圖呈現。 在[轉介流量](/help/dashboards/referral-traffic.md)儀表板上，使用內容傳遞網路記錄來顯示頁面點擊量度。 在內容傳遞網路記錄攝取或後續資料處理的任何階段，都不會處理或儲存任何個人識別資訊 (PII)。

## 篩選器 {#filters}

在頁面頂端，您可以套用篩選器來精確調整視圖。 您選擇的篩選器將影響儀表板上&#x200B;**所有**&#x200B;區段。 您可以自訂下列項目：

* **日期範圍**：選取所顯示資料的時間範圍。 例如最近 4 週。 您也可選取「**自訂週數**」選項來自訂時段。
* **類別**：依預先定義的類別或自訂類別篩選顯示的結果。
* **平台**：選擇要分析的 AI 引擎。
* **代理類型**：依照與您的網站互動的 AI 代理類型進行篩選。 您可以在爬蟲、聊天機器人或所有代理之間進行篩選。
* **成功率**：依互動品質 (高、中或低) 篩選。 這個量度代表成功的 HTTP 要求的百分比，包括直接成功回應 (2xx 狀態代碼) 以及重新導向 (3xx 狀態代碼)。
* **內容類型**：檢視不同內容類型 (例如 HTML、PDF 等) 的代理式互動。

選取想要的篩選器後，按一下「**套用篩選器**」，將選取項目套用至儀表板。

## 流量分佈 {#traffic-distribution}

流量分佈視圖會顯示代理式流量在不同的市場、類別和頁面類型之間的分佈方式。 因此，此視圖可協助您判斷 AI 代理與您的網站互動時，最常存取的地理位置、產品區域或內容格式。

![流量分佈](/help/dashboards/assets/ag-main.png)

您必須注意位在頁面頂端的三大主要量度：

* **代理式互動數**：此量度代表 AI 代理對您的網站提出的要求總數。 來自搜尋引擎、聊天機器人和其他非人類流量的所有流量皆包含在內。
* **成功率**：此量度代表成功的 HTTP 要求之百分比，包括直接成功回應和重新導向。
* **平均 TTFB**：第一個位元組時間 (TTFB) 測量收到伺服器發送的第一個位元組資料所需的時間。 平均值是根據傳回每種代碼的要求數量進行加權計算，並排除導致 5xx 回應的要求。 數值比較低表示伺服器回應時間較快。

每個主要量度的趨勢指標會顯示這些值在一段時間內與前一時段相比的變化。

## 代理式流量趨勢 {#agentic-trends}

使用代理式流量趨勢圖表來追蹤成功的、失敗的和整體點擊數的每週總計。 因此，您可以監視代理活動與效能在一段時間內的變化。 您也可以將滑鼠懸停在圖表上移動，查看每週時間範圍內的資料演變。

![代理式流量趨勢](/help/dashboards/assets/ag-trends.png)

## 上升最快者與下滑最快者 {#top-bottom-movers}

「上升最快者與下滑最快者」視圖會特別標示在代理式流量中週變動率最大的 URL，而代理式流量是指存取您內容的 AI 系統之瀏覽次數或點擊數。 **上升最快者**&#x200B;顯示可見度或參與度正在增長的頁面，而&#x200B;**下滑最快者**&#x200B;顯示跌幅最大的 URL。 您可以藉此快速確認哪些內容呈上升趨勢、哪些內容可能需要注意，以及 AI 驅動的探索模式往哪個方向轉變。

![上升最快者與下滑最快者](/help/dashboards/assets/movers.png)

## 使用者代理與 URL 效能分析 {#user-url-performance}

使用者代理和 URL 效能分析視圖提供進一步拆解的資料，解釋爬蟲和聊天機器人如何與您的網站互動。 點按下列分頁標籤以取得詳細說明。

![使用者代理與 URL 效能分析](/help/dashboards/assets/user-agent.png)

>[!BEGINTABS]

>[!TAB 使用者代理分析]

「使用者代理分析」表格依照頁面類型和代理類型 (例如爬蟲與聊天機器人) 劃分流量的組成。 您可以藉此輕鬆了解哪些 AI 代理正在抓取您網站的哪些部分。 表格中包含以下類別：

* **頁面類型**：該頁面的類型。
* **代理類型**：抓取頁面內容的 AI 代理，指爬蟲或聊天機器人。
* **點擊數**：AI 代理針對該特定頁面類型發出的要求總數。

您只要按一下「**設定欄**」按鈕，即可自訂要顯示的量度。

>[!TAB URL 效能分析]

「URL 效能分析」表格會顯示個別 URL 的詳細視圖， 其中包括點擊數、不重複的代理、熱門代理、成功率和類別。 您可以透過這些資訊來確認高價值的頁面、偵測抓取遺漏處，並最佳化內容在 AI 引擎的能見度。 URL 會依照流量排序。 此表格包含以下類別：

* **URL**：所檢查的 URL。
* **點擊總數**：AI 代理對該 URL 發出的要求總數。
* **不重複的代理**：曾存取該 URL 的不同的 AI 代理數量。
* **熱門代理**：對該 URL 帶來最多流量的 AI 代理。
* **熱門代理類型**：對該 URL 帶來最多流量的 AI 代理之類型。
* **成功率**：成功的 HTTP 要求的百分比，包括直接成功回應和重新導向。
* **類別**：與您的頁面內容最相符的類別。
* **平均 TTFB (ms)**：第一個位元組時間 (TTFB) 測量收到伺服器發出的第一個位元組資料所需的時間。 平均值是根據傳回每種代碼的要求數量進行加權計算，並排除導致 5xx 回應的要求。 數值比較低表示伺服器回應時間較快。
* **回應代碼**：針對該 URL 觀察到的 HTTP 狀態代碼。

URL 效能表格具有方便快速存取特定 URL 的搜尋欄位，而您只要按一下「**設定欄**」按鈕，即可自訂要顯示的量度。 您也可以按一下每列結尾的「**詳細資料**」圖示，檢視各個 URL 的其他詳細資料。

![URL 詳細資料](/help/dashboards/assets/details.png)

「URL 詳細資料」視圖從整體角度說明頁面的效能，顯示該 URL 被引用的頻率、提及該 URL 的 AI 回答的情緒、該 URL 出現在哪些主題和提示中，以及代理式和轉介流量在一段時間內的趨勢。

>[!ENDTABS]

您在這兩個表格上皆可使用「**匯出**」選項來下載 `.csv` 格式的表格，並與您的團隊共用洞察或將表格納入執行報告中。
