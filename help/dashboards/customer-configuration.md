# 客戶設定

「客戶組態」可讓您定義在LLM最佳化程式平台中如何監控及分析您的品牌。 您可以自訂類別（例如業務單位或產品線）、追蹤競爭者，以及新增品牌提及別名，以擷取不同提示中您品牌的所有變體。 此設定可確保平台根據您的業務內容量身打造深入分析，實現準確的可見度、流量和機會分析。

![客戶設定儀表板](/help/dashboards/assets/customer-config.png)


## 設定類別、主題和提示的最佳實務

本節說明決定如何設定類別、主題、提示和競爭者的最佳實務。

這是重要的第一步。 您現在所決定的內容，會決定如何針對您的業務內容量身打造資訊。 以後對類別所做的任何變更都會重設歷史資料。

### 類別的最佳實務

類別可讓您將內容組織到策略業務單位或邏輯群組中。 是「它所屬的位置」貯體，也是您內容的最上層組織結構。

在決定如何設定類別時，您需要考慮您的目標以及需要根據您的報告採取行動的對象。

>[!IMPORTANT]
>
> 確保類別從一開始就有正確的設定，因為對類別的變更會重設歷史資料。 這表示如果您變更這些專案，將會遺失變更前的歷史資料。

以下是您可採取之方法的型別概觀，以及選擇特定方法的時機：

| 方法 | 說明 | 優點 |
|---------|----------|---------|
| 策略業務單位(SBU) | 如果您的組織是以P&amp;L分割（例如，消費者、企業、服務），請使用此方法。 | 您可以依據業務線取得乾淨的報告，並更輕鬆負起責任。 |
| 網站最上層目錄(URL_DIR) | 如果您的網站資訊架構映象所有權(/products/、/support/、/docs/、/news/)，請使用。 | 您可以調整團隊發佈和維護內容的方式。 |
| 產品（或服務）類別 | 如果您銷售目錄（SKU/服務），請使用它。 | 您會獲得分類檢視、差距分析和「哪些類別需要內容」答案。 |

如何根據一個問題決定如何設定類別： **誰需要對報告採取行動？**

* 如果您是&#x200B;*業務領導者*，請選擇&#x200B;**SBU**&#x200B;方法。
* 如果您是&#x200B;*網頁/內容擁有者*，請選擇&#x200B;**URL_DIR**&#x200B;方法。
* 如果您是&#x200B;*銷售/優惠方案經理*，請選擇&#x200B;**產品/服務類別**&#x200B;方法。

>[!IMPORTANT]
>
> * 選擇一個方法並堅持它。
> * 每個帳戶/品牌只能有&#x200B;**一個**&#x200B;類別模型。 請勿同時混用&#x200B;**SBU**&#x200B;和&#x200B;**URL_DIR**。

範例：

| 網站型別 | 類別 | 主題分類法範例 |
|---------|----------|---------|
| 擁有多家企業的企業 | SBU | 小意向集（操作說明、疑難排解、比較、定價、政策） |
| 說明檔案/支援量大的網站 | URL_DIR | 操作說明，疑難排解，參考，發行說明 |
| 電子商務/服務目錄 | 產品/服務 | 比較、審查、定價/可用性、操作說明、疑難排解 |

### 主題的最佳實務

主題可協助您瞭解使用者意圖 — 向您顯示使用者想要什麼。 它們可讓您以類似的使用者意圖將提示分組。 將其視為將相關提示集結在一起。

>[!IMPORTANT]
>
>主題在&#x200B;**所有**&#x200B;類別中都是通用的，這表示無論它們被指派到哪個類別，它們都是一致的。 它們代表具有共同意圖的疑問或提示叢集。

決定主題時，您想要建立簡短且一般化的清單（最多6-12個）。 例如：

* 產品/服務
* 操作說明（設定/使用狀況）
* 疑難排解（錯誤/問題）
* 比較（X與Y；「最適合……」）
* 評論和評等
* 定價與可用性
* 原則與保固
* 支援聯絡人
* 企業/新聞（如果您真正需要的話）

建立清單時，請考量下列事項：

* 編輯者能否在5秒內從提示文字理解主題？ 如果沒有，請重新命名/簡化。
* 不同的團隊會擁有不同主題的修復嗎？ 如果有的話，您會挑選有用的主題。

其他有用的提示包括：

* 運用您的業務或網站知識來定義符合您品牌策略目標的主題
* 考慮特定主題中您的品牌與競爭對手的比較情形。

>[!IMPORTANT]
>
> * 保持主題以意圖為基礎，而不是以組織為基礎。
> * 請勿新增品牌/非品牌/地理的類別/篩選器，因為您可以在「**品牌**」標籤中為此特別進行篩選。
> * 主題分散在數個類別中，您&#x200B;**不能**&#x200B;每個類別有不同的主題。
> * 單一提示可以存在於多個主題或類別中。

### 提示的最佳實務

提示可識別客戶提出的特定問題或查詢，這些問題或查詢可能會影響您的業務。 它們是使用者輸入到LLM中的實際問題或查詢。

請務必定期檢閱和更新提示，以確保符合客戶需求和業務目標。

>[!TIP]
>
>* 您可以使用如LLM Optimizer和Google Search Console等工具，搭配規則運算式篩選器，以識別常見的問題結構（例如，「如何」、「什麼」、「何時」、「何處」）。
>* 若要瞭解哪些提示與您的網站/品牌相關，您可以使用網站上的搜尋資料、搜尋引擎結果頁面中的常見問題集，甚至直接向LLM聊天機器人詢問客戶可能對您的品牌提出的疑問。

### 適用於競爭對手的最佳實務

競爭者可讓您監控LLM回應中對於您的企業而言重要提示和主題的可見度和提及內容。

[!UICONTROL **競爭者追蹤**]&#x200B;索引標籤可讓您新增競爭者，並追蹤其特定提示和主題的可見度。

透過競爭者追蹤，您可以瞭解競爭者在不同地區和類別中與您的品牌一起被提及的頻率，並將其可見度與您自己的可見度進行比較。

>[!TIP]
>
>定期檢閱競爭者提及和引文，以找出您的品牌可以改善的領域。

## 客戶設定儀表板

「客戶設定儀表板」是一個功能強大的工具，可讓您深入瞭解品牌在LLM中的可見度。 透過正確設定類別、主題、提示和競爭者，您可以確保您的品牌在LLM產生的回應中處於良好的位置。 定期檢閱如分享聲音、內容可見度和機會等見解，可幫助您調整策略並保持領先於競爭對手。

為了設定LLM Optimizer如何跨不同市場和競爭環境監視和分析您的品牌存在，您可以存取以下標籤：

* [類別](#categories)
* [競爭者追蹤](#competitor-tracking)
* [品牌別名](#brand-aliases)
* [資料分析](#data-insights)
* [代理CDN](#agentic-cdn)

## 類別 {#categories}

從類別頁標，您可以定義要追蹤的業務類別或產品線，並將其與特定區域建立關聯。 整體而言，類別標籤會與此頁面上的其他每個自訂專案相關，因為其他自訂專案（競爭者追蹤、別名等）的類別欄位中會顯示類別。 若要新增類別：

1. 按一下&#x200B;**新增**&#x200B;按鈕。
2. 在新設定視窗中，新增&#x200B;**類別名稱**。
3. 自訂將監視類別的&#x200B;**關聯區域**。
4. 按一下&#x200B;**儲存**，新類別就會出現在類別清單中。

新增類別不會自動產生主題和提示，這些主題和提示需要從[資料深入分析](#data-insights)索引標籤手動新增。

若要刪除類別，請按一下類別清單中的刪除圖示。 請小心，因為&#x200B;**刪除類別也會刪除關聯的專案**，例如您可能已設定的競爭者或連結至該特定類別的品牌別名。

## 競爭者追蹤 {#competitor-tracking}

透過使用競爭者追蹤，您可以跨不同類別和區域追蹤與您的品牌相關的競爭者提及方式。 監控其在您市場區段中的現況和效能。 若要自訂競爭者追蹤：

1. 若要新增競爭者，請按一下[新增]按鈕。****
2. 在新的設定視窗中，選取&#x200B;**類別**。 先前建立的類別將顯示在這裡。
3. 新增競爭者的名稱。
4. 視需要自訂競爭者的「別名」和「網域」。
5. 按一下「儲存」****，新的競爭者就會出現在競爭者清單中。

若要刪除競爭者，請按一下競爭者清單中的刪除圖示。

## 品牌別名 {#brand-aliases}

透過使用品牌別名，您可以設定應跨不同類別和區域追蹤的替代名稱和品牌變體。 這可確保全面監控所有品牌提及。 若要新增品牌別名：

1. 按一下&#x200B;**新增**&#x200B;按鈕。
2. 在新的設定視窗中，選取&#x200B;**類別**。 先前建立的類別將顯示在這裡。
3. 選取要監視別名的&#x200B;**區域**。
4. 新增品牌別名。
5. 按一下&#x200B;**儲存**，品牌別名就會出現在清單中。

若要刪除品牌別名，請按一下別名清單中的刪除圖示。

## 資料分析 {#data-insights}

您可以從此標籤檢閱、管理及自訂提示。 您可以上傳[Brand Presence資料深入分析](/help/dashboards/brand-presence.md#data-insights) .csv，清單中會填入該分析的提示和主題。 您也可以視需要刪除、修改和新增主題及其相關提示。

若要匯入資料深入分析.csv檔案，您必須先從Brand Presence儀表板匯出檔案。 請參閱[資料深入分析](/help/dashboards/brand-presence.md#data-insights)章節以瞭解如何操作。 取得檔案後：

1. 在儀表板上，按一下&#x200B;**上傳CSV**。
2. 在匯入資料深入分析視窗上，拖放或手動選擇檔案。
3. 按一下&#x200B;**上傳資料**。

您也可以從&#x200B;**匯入資料分析**&#x200B;視窗下載範本，以建立新的CSV檔案。 取得範本後，請開啟範本，並以新行輸入主題及其相關的提示、類別和區域。

此外，您也可以在CSV檔案以外單獨將主題/提示新增至清單。 若要完成此操作，在控制面板上，您需要：

1. 按一下&#x200B;**新增主題**&#x200B;按鈕。
2. 在新的設定視窗中，選取&#x200B;**類別**。 先前建立的類別將顯示在這裡。
3. 輸入主題名稱。
4. 新增提示文字。
5. 選取地區。
6. 按一下&#x200B;**新增提示**，含有提示的主題就會出現在清單中。

>[!NOTE]
>處理完成前，新新增的提示不會出現在品牌顯示中。

在清單中，您可以按一下每個主題，相關的提示就會顯示。若要刪除主題及其相關的提示，請按一下清單中的刪除圖示。
