# 客戶設定 {#customer-configuration}

「客戶設定儀表板」是功能強大的工具，可協助您洞察品牌在 LLM 中的能見度。 只要正確設定類別、主題、提示，便可以確保品牌具備出現在 LLM 生成回答的極佳條件。 此設定可確保平台根據您的業務情境提供量身打造的洞察，讓您能夠準確地分析能見度、流量和機會。

![客戶設定儀表板](/help/dashboards/assets/customer-config.png)

您擁有以下分頁標籤的存取權，以便設定 LLM Optimizer 如何針對不同的市場和競爭態勢監視及分析您的品牌存在感：

* [提示](#prompts-brand)
* [類別](#categories)
* [其他品牌](#other-brands)
* [品牌別名](#brand-aliases)
* [內容傳遞網路設定](#agentic-cdn)
* [Google Search Console](#google-console)

>[!IMPORTANT]
>
> 如需如何設定類別、主題、提示的詳細資訊，請參閱[設定類別、主題、提示的最佳做法](/help/overview/best-practices-topics-prompts.md)頁面。

## 提示 {#prompts-brand}

透過這個分頁標籤，您可以審閱、管理及自訂提示。 您可以上傳[品牌存在感分析](/help/dashboards/brand-presence.md) .csv 檔案並從該分析中將提示和主題填入清單，或者[下載由 Adobe 建立的提示資料庫](/help/overview/best-practices-topics-prompts.md)。 您也可以按照需要刪除、修改和新增主題及其相關提示。

若要匯入資料洞察 .csv 檔案，您必須先從品牌存在感儀表板匯出檔案。 請參閱[資料洞察](/help/dashboards/brand-presence.md#data-insights)區段以了解操作方式。 在您取得檔案後：

1. 在儀表板上按一下「**上傳 CSV**」。
2. 在「匯入資料洞察」視窗上，拖放或手動選擇檔案。
3. 按一下「**上傳資料**」。

您也可以從「**匯入資料洞察**」視窗下載範本，並建立新的 CSV 檔案。 取得範本之後，請開啟範本並輸入您的主題及其關聯的提示、類別和區域，各為一列。

若要了解如何下載和使用由 Adobe 建立的產業提示資料庫，請參閱[本頁](/help/overview/best-practices-topics-prompts.md)的產業提示資料庫區段

此外，您無需透過 CSV 檔案或提示資料庫，也可以獨立在清單中加入主題或提示。 若要完成這項操作，您必須在儀表板上：

1. 按一下「**新增主題**」按鈕。
2. 在新的設定視窗中，選取「**類別**」。 先前建立的類別將顯示在這裡。
3. 輸入主題名稱。
4. 新增提示文字。
5. 選取區域。
6. 按一下「**新增提示**」，然後含有提示的主題就會出現在清單中。

>[!NOTE]
>新增加的提示在處理完成前，不會顯示在「品牌存在感」中。

您可以在清單上按一下每個主題，相關的提示就會顯示。若要刪除主題及其相關的提示，請按一下清單中的刪除圖示。

## 類別 {#categories}

您可以在類別分頁標籤中定義要追蹤的業務類別或產品線，並將其與特定區域建立關聯。 總體而言，類別分頁標籤與此頁面上的幾乎所有其他自訂項目皆相關，因為類別會出現在其他自訂項目 (其他追蹤、別名等) 的類別欄位中。 若要新增類別：

1. 按一下「**新增**」按鈕。
2. 在新設定視窗中，新增「**類別名稱**」。
3. 自訂要監視該類別的「**相關聯的區域**」。
4. 按一下「**儲存**」，新類別便會出現在類別清單中。

新增類別並不會自動生成主題和提示，這些主題和提示必須透過「[資料洞察](#data-insights)」分頁標籤手動新增。

若要刪除類別，請按一下類別清單中的刪除圖示。 請小心，因為&#x200B;**刪除類別也會刪除相關聯的項目**，例如與該特定類別連結的品牌別名。

## 其他品牌 {#others-tracking}

您可以利用此分頁標籤來追蹤其他品牌在不同的類別和區域中，在與您的品牌關聯下被提及的情況。 在您的細分市場中監控其存在感與表現。 若要自訂追蹤：

1. 按一下「**新增**」按鈕。
2. 在新的設定視窗中，選取「**類別**」。 先前建立的類別將顯示在這裡。
3. 新增其他品牌的名稱。
4. 視需要自訂其他別名與網域。
5. 按一下「**儲存**」。

若要刪除清單上的項目，請按一下刪除圖示。

## 品牌別名 {#brand-aliases}

您可以使用品牌別名，針對應在不同類別和區域中進行追蹤的您的品牌，設定替代名稱和變化版本。 這樣可確保全面追蹤所有品牌提及的情形。 若要新增品牌別名：

1. 按一下「**新增**」按鈕。
2. 在新的設定視窗中，選取「**類別**」。 先前建立的類別將顯示在這裡。
3. 選取要監視其別名的「**區域**」。
4. 新增品牌別名。
5. 按一下「**儲存**」，品牌別名便會出現在清單上。

若要刪除品牌別名，請按一下別名清單中的&#x200B;**刪除**&#x200B;圖示。

## 內容傳遞網路設定 {#cdn-configuration}

您可以利用此分頁標籤來設定內容傳遞網路串流，讓 Adobe LLM Optimizer 分析您的內容傳遞網路資料。 這些資訊將可用於支援儀表板 (如代理式流量)，提供有關流量模式、效能量度和最佳化機會的洞察。 若要讓您的內容傳遞網路供應商上線，請點選「**內容傳遞網路上線**」。

![客戶設定內容傳遞網路](/help/overview/assets/cc-cdn.png)

在「**內容傳遞網路提供者上線**」視窗上：

1. 選取您的內容傳遞網路提供者。
2. 按一下「**上線**」以啟用記錄轉送。

如果您選取「**其他**」，則必須聯絡 llmo-now@adobe.com 以取得協助。

## Google Search Console {#google-console}

Adobe LLM Optimizer可讓您整合Google Search Console帳戶，將真正的搜尋查詢直接帶入介面。 透過顯示真實的Google Search Console查詢，您可以建立以實際搜尋行為和高意圖探索模式為基礎的提示集。 這可協助您根據證實的需求來排定提示的優先順序，並將LLM最佳化工作與使用者目前搜尋的方式保持一致。 此外，您仍可完全控制，因為查詢不會自動新增，且必須在成為使用中提示之前明確選取。

### 運作方式 {#how-it-works}

關於LLM Optimizer與Google搜尋主控台之間的整合，要記住的主要事項如下：我們不以手動方式猜測客戶可能會向AI助理詢問哪些內容，而是檢視他們&#x200B;**已在搜尋**&#x200B;哪些內容，並將這些真正的查詢轉換為自然、對話式的提示。 從搜尋查詢移動到AI提示的這個過程在下圖中舉例說明。

![處理流程](/help/dashboards/assets/diagram-flow.png)

一般而言，這個程式有五個步驟：

#### 步驟1 — 收集您實際的搜尋資料 {#gsc-one}

此程式會從您的對象透過Google找到您的網站時實際使用的關鍵字開始。 這個原始資料集（通常有數千個不重複查詢）是後續工作的基礎。

#### 步驟2 — 分析意義並篩選安全性 {#gsc-two}

系統會分析每個查詢的語意意義（使用者真正想知道的是什麼），並透過安全篩選條件進行篩選，以移除不適當或品牌外內容。 這樣可確保只推出簡潔的相關關鍵字。

#### 步驟3 — 分組為類別和主題 {#gsc-three}

相關查詢會自動分組為&#x200B;**類別** （廣泛的業務主題）和&#x200B;**主題** （每個類別中重點的子主題）。 系統會優先處理已在LLM Optimizer設定中設定的類別。 此外，它也會顯示您的搜尋資料所揭示但尚未監視的新類別。 下圖是傢俱品牌的類別和主題範例：

![傢俱品牌](/help/dashboards/assets/diagram-example.png)

#### 步驟4 — 以實際關鍵字為基礎產生提示 {#gsc-four}

對於每個主題，系統都會產生提示，類似於真人與AI助理交談的方式。 每個提示都直接受到您Google搜尋控制檯中的實際搜尋關鍵字影響，將關鍵字意圖轉換為自然的對話式問題。

此方法（根據關鍵字）表示：

* 提示反映的是真實的需求，而不是假設的問題。
* 語言反映了客戶實際用詞的方式。
* 涵蓋範圍涵蓋使用者在您的網站上搜尋的所有專案。

提示產生時也會考量您的品牌設定檔 — 包括產品、競爭者、產業定位和目標對象 — 以確保提示內容準確。

#### 步驟5 — 品質保證與交付 {#gsc-five}

在傳送前，每個提示都會經過數次自動化品質檢查：

* 去重複化 — 移除幾乎相同的提示。
* 品牌比例平衡 — 確保實際混合（約75%未品牌，約25%品牌）。
* 語言品質 — 去除機械式短語，以提示聲音自然。
* 一致性檢查 — 驗證日期、移除填寫片語、確保簡潔的長度。

此外，每個提示都標示其類別、主題、意圖型別和品牌/非品牌分類，讓LLM Optimizer可以開始監視。

#### 提示剖析 {#prompt-anatomy}

完成上述程式後，傳送至LLM Optimizer的每個提示都會具備下列屬性：

| 欄位 | 說明 |
|---------|----------|
| 文字 | 提示，類似於使用者將其輸入到AI助理中的方式 |
| 類別 | 指派給此提示的廣泛業務主題。 |
| 主題 | 類別中的特定子主題。 |
| 區域 | 目標市場（例如美國、英國等）。 |
| 意圖 | 使用者心態：資訊、比較、異動、指示、規劃或委派。 |
| 類型 | 型別可以是品牌化（提及品牌/產品）或非品牌化（一般產業問題）。 |

### 使用方法 {#how-to-use}

請依照下列步驟，將Google Search Console查詢與LLM Optimizer整合及使用。

#### 連線Google搜尋主控台 {#connect-console}

使用此功能之前，您需要將Google Search Console帳戶與LLM最佳化工具整合。

1. 開啟Customer Configuration控制面板。
1. 瀏覽至Google Search Console索引標籤，然後按一下&#x200B;**連線帳戶**。
   ![Google搜尋主控台](/help/dashboards/assets/google-console.png)
1. 使用可存取所需「搜尋主控台」屬性的Google帳戶登入。
   ![Google帳戶](/help/dashboards/assets/google-account.png)
1. 選擇您要連線的屬性。
   ![主控台屬性](/help/dashboards/assets/console-property.png)
1. 連線完成後，LLM Optimizer會開始擷取相關的搜尋查詢。
   ![正在擷取資料](/help/dashboards/assets/console-complete.png)

#### 檢閱和搜尋查詢 {#search-query}

將Google Search Console帳戶與LLM最佳化工具整合後，您可以檢閱源自搜尋主控台的主題和提示清單，並從清單新增提示。

1. 在Google「搜尋主控台」索引標籤上，檢閱來源為「搜尋主控台」的主題和提示清單。
   ![提示清單](/help/dashboards/assets/prompts-list.png)
1. 按一下所需的主題/提示類別以展開清單。
1. 使用&#x200B;**新增**&#x200B;按鈕從清單新增提示。 您也可以使用&#x200B;**全部新增**&#x200B;來大量新增提示和類別。
   ![新增提示](/help/dashboards/assets/add-prompts.png)
1. 在您滿意選取範圍後，請按一下通知訊息上的[儲存]。**&#x200B;**

#### 檢視在提示清單中新增的查詢 {#prompts-list}

新增查詢後，該查詢會顯示在「客戶設定」儀表板的[提示](#prompts-brand)索引標籤中。 來自Google搜尋主控台的提示在&#x200B;**Origin**&#x200B;欄中會以Google搜尋主控台圖示標籤。 圖示可協助您區別以實際使用者搜尋行為為基礎的提示，與手動新增的提示或其他來源的提示。

### 常見問題 {#gsc-faq}

問：在Google Search Console儀表板中更新提示的頻率為何？

來自Google Search Console的提示通常每月會重新整理一次。 每次重新整理都會從Google搜尋控制檯提取最新搜尋查詢資料、重新執行產生管道，並更新提示集。 這可確保您的提示與目前的搜尋趨勢和使用者行為的季節性轉變保持一致。

問：通常有多少提示源自Google搜尋主控台？

數目取決於部署的大小和追蹤的類別數量。 例如：

| 類別 | 主題總數 | 提示已傳送 |
|---------|----------|----------|
| 1-2 | 3-8 | ~65-180 |
| 4-5 | 12-20 | ~270-450 |
| 10 | 30-40 | ~675-900 |

我們的目標是提供符合在試用和上線期間所傳達的品質目標的提示集：每個主題至少20個提示，每個類別有3到4個主題，以及健康的品牌/品牌平衡。

問：在連線至Google搜尋主控台後，我多久會看到來自Google搜尋主控台的提示？

建立您的Google Search Console連線後，通常會在數小時&#x200B;**內**&#x200B;提供提示。 該管道會自動提取您的搜尋資料，經過產生和品質保證步驟加以處理，並將最終提示集傳送給LLM Optimizer。

問：誰可以連線至Google Search Console？

任何擁有Google Search Console屬性上的&#x200B;**擁有者**&#x200B;或&#x200B;**完整許可權**&#x200B;的人都可以授權連線。 這些許可權層級可授予搜尋查詢資料的讀取存取權。 如果您不確定您的許可權等級，可以在Google Search Console的&#x200B;**設定>使用者**&#x200B;和許可權下檢視它。

問：我是否可以將提示標示為已忽略或已略過，以便在Google Search Console提示清單中看不到？

可以，您可以刪除不想監視的任何提示。 已刪除的提示會從您使用中的提示清單中移除，且不會出現在未來的報告中。 如果刪除的提示在後續的每月重新整理中重新產生，您可以再次移除它。

問：一旦我從Google搜尋控制檯新增提示至提示清單後，多久才能看到這些提示的品牌存在感資料？

新新增提示的品牌存在感資料會在下次排程資料重新整理時顯示，通常在每週初執行。 根據您新增提示的時間，您可能會在幾天內看到結果。
