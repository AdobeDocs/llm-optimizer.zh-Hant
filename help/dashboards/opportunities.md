# 最佳化機會

最佳化機會是自動偵測的深入分析，顯示您的網站和外部存在可以改善到何處，以提高AI搜尋中的品牌可見度。 這些最佳化包括頁面上的修正（新增結構化內容、規範或摘要）、技術調整（解除封鎖AI編目程式或解決錯誤）以及影響第三方權威網站上的內容。 把握這些最佳化機會有助於準確表示您的品牌，且更有可能在產生式回應中被引用。

![最佳化機會](/help/dashboards/assets/oport.png)

## 機會控制面板

控制面板上顯示的最佳化機會會根據播放器差距、趨勢主題和效能資料來排定優先順序，並以清單顯示。 您可以使用搜尋欄位來搜尋特定商機。 此外，商機會依標籤分組，您可以直接按一下標籤以顯示該標籤下分組的所有商機。

按一下「**詳細資料**」會開啟另一個視窗，其中會提供詳細資訊和其他指引。

## 支援的機會

下表列出目前支援的機會：

| 機會 | 類型 | 識別的問題 | 修正建議 |
|---------|----------|----------|----------|
| 摘要長段落 | 內容（站上） | 偵測超過建議長度臨界值的段落。 顯示受影響的URL和過大的文字片段。 | 建立摘要或將長文字分割成較短、可掃描的區段。 |
| 建議的結構化內容(FAQ) | 內容（站上） | 偵測沒有相符常見問答集專案的高人氣提示。 顯示相關提示、類別和受影響的URL。 | 新增具有簡明答案的常見問題集結構描述區塊，以符合常見查詢。 |
| 偵測遺失的Hreflang | 內容（站上） | 標示缺少hreflang屬性的頁面。 依語言/地區提供受影響的URL和預期的涵蓋範圍。 | 實作hreflang標籤以指出正確的本地化版本。 |
| 偵測遺失的規範 | 內容（站上） | 偵測超過建議長度臨界值的段落。 顯示受影響的URL和過大的文字片段。 | 建立摘要或將長文字分割成較短、可掃描的區段。 |
| 偵測空白標題 | 內容（站上） | 標示標題標籤存在但不包含文字的頁面。 顯示空白標籤的URL和位置。 | 將描述性文字新增至反映其下內容的標題。 |
| 偵測重複的標題 | 內容（站上） | 掃描HTML標題標籤並標示重複的標題。 顯示受影響的URL和重複的文字片段。 | 修訂標題以使其成為唯一併維護階層(H1 → H2 → H3)。 合併或重新命名重複的區段。 |
| 偵測封鎖的代理程式流量 | 技術地理位置 | 分析CDN記錄檔中來自已知AI代理程式（例如GPTBot、PerplexityBot）的封鎖請求。 報告受影響的URL和代理程式。 | 更新robots.txt或伺服器設定，以便在適當時允許受支援的AI編目程式存取。 |
| 偵測404s / 403s / 5xx問題 | 技術地理位置 | 監視CDN記錄檔的錯誤回應。 報表頻率、受影響的URL以及預估的點選遺失。 | 修正中斷的連結、更新許可權，並解決伺服器端問題，使關鍵內容傳回200個回應。 |
