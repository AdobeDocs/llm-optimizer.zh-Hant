# 在這裡介紹

「客戶組態」可讓您定義在LLM最佳化程式平台中如何監控及分析您的品牌。 您可以自訂類別（例如業務單位或產品線）、追蹤競爭者，以及新增品牌提及別名，以擷取不同提示中您品牌的所有變體。 此設定可確保平台根據您的業務內容量身打造深入分析，實現準確的可見度、流量和機會分析。

## 設定類別、主題和提示的最佳實務

本節說明決定如何設定類別、主題、提示和競爭者的最佳實務。
這是重要的第一步。 您現在所決定的內容，會決定如何針對您的業務內容量身打造資訊。 以後對類別所做的任何變更都會重設歷史資料。

### 類別的最佳實務

類別可讓您將內容組織到策略業務單位或邏輯群組中。 是「它所屬的位置」貯體，也是您內容的最上層組織結構。

在決定如何設定類別時，您需要考慮您的目標以及需要根據您的報告採取行動的對象。

>[!IMPORTANT]
>
> 確保類別從一開始就有正確的設定，因為對類別的變更會重設歷史資料。 這表示如果您變更這些專案，將會遺失變更前的歷史資料。

以下是您可採取之方法的型別概觀，以及選擇特定方法的時機：

| 方法 | 說明 | 優點 |
|---------|----------|---------|
| 策略業務單位(SBU) | 如果您的組織是以P&amp;L分割（例如，消費者、企業、服務），請使用此方法。 | 您可以依據業務線取得乾淨的報告，並更輕鬆負起責任。 |
| 網站最上層目錄(URL_DIR) | 如果您的網站資訊架構映象所有權(/products/、/support/、/docs/、/news/)，請使用。 | 您可以調整團隊發佈和維護內容的方式。 |
| 產品（或服務）類別 | 如果您銷售目錄（SKU/服務），請使用它。 | 您會獲得分類檢視、差距分析和「哪些類別需要內容」答案。 |

如何根據一個問題決定如何設定類別： **誰需要對報告採取行動？**

* 如果您是&#x200B;*業務領導者*，請選擇&#x200B;**SBU**&#x200B;方法。
* 如果您是&#x200B;*網頁/內容擁有者*，請選擇&#x200B;**URL_DIR**&#x200B;方法。
* 如果您是&#x200B;*銷售/優惠方案經理*，請選擇&#x200B;**產品/服務類別**&#x200B;方法。

>[!IMPORTANT]
>
> * 選擇一個方法並堅持它。
> * 每個帳戶/品牌只能有&#x200B;**一個**&#x200B;類別模型。 請勿同時混用&#x200B;**SBU**&#x200B;和&#x200B;**URL_DIR**。

範例：

| 網站型別 | 類別 | 主題分類法範例 |
|---------|----------|---------|
| 擁有多家企業的企業 | SBU | 小意向集（操作說明、疑難排解、比較、定價、政策） |
| 說明檔案/支援量大的網站 | URL_DIR | 操作說明，疑難排解，參考，發行說明 |
| 電子商務/服務目錄 | 產品/服務 | 比較、審查、定價/可用性、操作說明、疑難排解 |

### 主題的最佳實務

主題可協助您瞭解使用者意圖 — 向您顯示使用者想要什麼。 它們可讓您以類似的使用者意圖將提示分組。 將其視為將相關提示集結在一起。

>[!IMPORTANT]
>
>主題在&#x200B;**所有**&#x200B;類別中都是通用的，這表示無論它們被指派到哪個類別，它們都是一致的。 它們代表具有共同意圖的疑問或提示叢集。

決定主題時，您想要建立簡短且一般化的清單（最多6-12個）。 例如：

* 產品/服務
* 操作說明（設定/使用狀況）
* 疑難排解（錯誤/問題）
* 比較（X與Y；「最適合……」）
* 評論和評等
* 定價與可用性
* 原則與保固
* 支援聯絡人
* 企業/新聞（如果您真正需要的話）

建立清單時，請考量下列事項：

* 編輯者能否在5秒內從提示文字理解主題？ 如果沒有，請重新命名/簡化。
* 不同的團隊會擁有不同主題的修復嗎？ 如果有的話，您會挑選有用的主題。

其他有用的提示包括：

* 運用您的業務或網站知識來定義符合您品牌策略目標的主題
* 考慮特定主題中您的品牌與競爭對手的比較情形。

>[!IMPORTANT]
>
> * 保持主題以意圖為基礎，而不是以組織為基礎。
> * 請勿新增品牌/非品牌/地理的類別/篩選器，因為您可以在「**品牌**」標籤中為此特別進行篩選。
> * 主題分散在數個類別中，您&#x200B;**不能**&#x200B;每個類別有不同的主題。
> * 單一提示可以存在於多個主題或類別中。

### 提示的最佳實務

提示可識別客戶提出的特定問題或查詢，這些問題或查詢可能會影響您的業務。 它們是使用者輸入到LLM中的實際問題或查詢。

請務必定期檢閱和更新提示，以確保符合客戶需求和業務目標。

>[!TIP]
>
>* 您可以使用如Adobe LLM Optimizer和Google Search Console等工具，搭配規則運算式篩選器，識別常見的問題結構（例如，「如何」、「什麼」、「何時」、「何處」），並找出是什麼會提示人們使用造訪您的網站。
>* 若要瞭解哪些提示與您的網站/品牌相關，您可以使用網站上的搜尋資料、搜尋引擎結果頁面中的常見問題集，甚至直接向LLM聊天機器人詢問客戶可能對您的品牌提出的疑問。

### 適用於競爭對手的最佳實務

競爭者可讓您監控LLM回應中對於您的企業而言重要提示和主題的可見度和提及內容。

[!UICONTROL **競爭者追蹤**]&#x200B;索引標籤可讓您新增競爭者，並追蹤其特定提示和主題的可見度。

透過競爭者追蹤，您可以瞭解競爭者在不同地區和類別中與您的品牌一起被提及的頻率，並將其可見度與您自己的可見度進行比較。

>[!TIP]
>
>定期檢閱競爭者提及和引文，以找出您的品牌可以改善的領域。
